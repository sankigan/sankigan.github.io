LCR187.破冰游戏

sankigan2026-6-30算法LeetCode

破冰游戏open in new window

题目描述

社团共有 num 名成员参与破冰游戏，编号为 0 ~ num-1。成员们按照编号顺序围绕圆桌而坐。社长抽取一个数字 target，从 0 号成员起开始计数，排在第 target 位的成员离开圆桌，且成员离开后从下一个成员开始计数。请返回游戏结束时最后一位成员的编号。

示例

输入：num = 7, target = 4
输出：1

输入：num = 12, target = 5
输出：0

解答

数学递推

这是经典的约瑟夫环问题：N 个人围成一圈，从 1 开始报数，报到 M 的人被淘汰，下一位从 1 重新开始，求最后剩下的人。暴力模拟每轮线性定位被淘汰者，整体 O(n²)；利用数学递推可以压到 O(n)。

关键思路

全文最重要的一点：只关心最终活着那个人的索引号变化，不用追踪每个人的生死。

设 f(n, target) 表示 n 人、每次报到 target 的游戏中，最终胜者（活下来的人）的索引号。

每次淘汰一人后，圆圈缩小为 n - 1 人，从下一位开始重新编号 0, 1, 2, ...。如果已知 n - 1 人游戏的胜者在新圆圈中的索引 f(n - 1, target)，只需把它"映射回"原 n 人圆圈，就能得到 f(n, target)。

编号变化（图解）

很多题解用数字举例，看完仍有疑惑。这里换个角度，给每个人一个字母编号，以 N = 8、target = 3 为例，只观察最终活下来的 G 的索引变化：

从 8 个人开始，每次杀掉一个人，去掉被杀者，然后把其后第一个人作为开头重新编号：

轮次被杀剩余人数新开头G 的索引变化
1C7D6 → 3
2F6G3 → 0
3A5B0 → 3
...............
末轮-1-必定为 0

重点：当只剩一人时，他的索引必定为 0，即 f(1, target) = 0。

反推递推式

知道了 G 的索引变化过程，反过来推：如何从 N = 7 的排列恢复回 N = 8？

做法是先把被杀掉的 C 补回来，然后把胜者整体右移 target 位，溢出的部分回到队首，就回到了 N = 8 的排列：

因此 f(8, 3) = [f(7, 3) + 3] % 8，推广到一般情况：

f(n, target) = [f(n - 1, target) + target] % n

再补上边界条件 n = 1，得到完整的递推公式：

f(n, target) = 0,                                n = 1
f(n, target) = [f(n - 1, target) + target] % n,  n > 1

从 i = 2 迭代到 i = n，每轮用上一结果加 target 取模即可，无需额外空间。

复杂度

  • 时间：O(n)，线性迭代一遍。
  • 空间：O(1)，只用一个变量滚动更新。
Code
function iceBreakingGame(num: number, target: number): number {
  let res = 0;
  for (let i = 2; i <= num; ++i) {
    res = (res + target) % i;
  }
  return res;
}

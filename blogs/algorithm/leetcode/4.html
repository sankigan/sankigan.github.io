4.寻找两个正序数组的中位数

sankigan2026-6-29算法LeetCode

寻找两个正序数组的中位数open in new window

题目描述

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的中位数。

算法的时间复杂度应该为 O(log (m+n)) 。

示例

输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2

输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5

提示

  • nums1.length == m
  • nums2.length == n
  • 0 <= m <= 1000
  • 0 <= n <= 1000
  • 1 <= m + n <= 2000
  • -10^6 <= nums1[i], nums2[i] <= 10^6

解答

解法一：归并双指针（O(m + n)）

关键思路

最直观的思路：既然两个数组都已经有序，那就用归并排序的「合并」步骤把它们拼成一个新有序数组，再按下标取中位数。

  1. 用双指针 p1、p2 分别指向 nums1、nums2 的当前最小未取元素；
  2. 每轮比较两指针所指的值，把较小者写入合并数组的 p1 + p2 位置，对应指针前进一位；
  3. 任一数组耗尽后，把另一数组的剩余元素直接拷贝到末尾；
  4. 总长度 len = m + n 的中位数有两种情况：
    • 奇数：取下标 (len - 1) / 2 处的元素；
    • 偶数：取下标 len / 2 - 1 和 len / 2 两个元素的均值。

技巧：令 mid = (len - 1) / 2，取 low = floor(mid) 与 high = ceil(mid)，返回 (arr[low] + arr[high]) / 2。这样奇偶可以统一表达——奇数时 low === high，相当于该元素自身；偶数时 low 和 high 是中间两个下标。

复杂度

  • 时间：O(m + n)，每个元素入合并数组一次。
  • 空间：O(m + n)，合并数组。

该解法不满足题目进阶的 O(log(m + n)) 要求，但思路清晰、易写易调试，适合作为兜底。

Code
function findMedianSortedArrays(nums1: number[], nums2: number[]): number {
  const m = nums1.length;
  const n = nums2.length;
  const arr: number[] = new Array(m + n);
  let p1 = 0;
  let p2 = 0;

  while (p1 < m || p2 < n) {
    // 一侧耗尽，另一侧整段补完后退出
    if (p1 === m) {
      while (p2 < n) arr[p1 + p2] = nums2[p2++];
      break;
    }
    if (p2 === n) {
      while (p1 < m) arr[p1 + p2] = nums1[p1++];
      break;
    }

    // 取两指针所指较小者写入合并位置
    if (nums1[p1] < nums2[p2]) {
      arr[p1 + p2] = nums1[p1++];
    } else {
      arr[p1 + p2] = nums2[p2++];
    }
  }

  // 奇偶统一: low === high 时退化为单元素自身
  const mid = (arr.length - 1) / 2;
  const low = Math.floor(mid);
  const high = Math.ceil(mid);
  return (arr[low] + arr[high]) / 2;
}

解法二：划分二分（O(log(m + n))）

关键思路

把"求中位数"转化为"求第 k 小"：奇数取第 (m + n + 1) / 2 小，偶数取第 (m + n) / 2 与第 (m + n) / 2 + 1 小的均值。

问题变成：如何在不合并数组的情况下快速找到第 k 小？ 答案是每轮排除掉一批肯定不是答案的元素，让 k 不断减半。

核心：为什么两侧各取 k / 2 个，且可以安全排除较小的那侧？

以 k = 6 为例（用 A、B 代两数组）：

A: [ 1,  3,  5 | ... ]   ← A[2] = 5（候选，较小）
B: [ 2,  4,  9 | ... ]   ← B[2] = 9（候选，较大）

两侧各取第 k / 2 = 3 个，比较末尾：A[2] = 5 < B[2] = 9。

此时 A[2] 在合并数组里的排名最多是第几？

  • A 里 ≤ 5 的：3 个（A[0..2]）；
  • B 里 ≤ 5 的：最多 2 个（B[2] = 9 > 5，所以 B[2] 及之后都不算）；
  • 合计排名上界：3 + 2 = 5 < 6 ✅

A[2] 最多是第 5 小，绝不可能是第 6 小；A[0]、A[1] 比它更小，同样排除。A[0..2] 整段安全删掉，问题缩减为「在 A[3..] 与 B 中找第 k - 3 小」。

推广到通式：较小候选侧贡献 k/2 个，较大候选侧最多贡献 k/2 - 1 个，排名上界 = k - 1 < k，永远不会误删答案。每轮 k 至少减半，复杂度 O(log k)。

为什么不能排 B 那侧？把 B 改成 [2, 4, 6, 7, 8]，合并后第 6 小恰好是 B[2] = 6。较大那侧里可能藏着答案，不能动。

为什么 k 用 1-Based？k 表示"第几小"，是计数语义。统一 1-Based 后，淘汰量直接写 k -= half，单侧定位写 nums[idx + k - 1]，-1 只在最终读元素时出现一次；若改用 0-Based，取候选、淘汰量、终止条件处处多一个 ±1 修正，容易写错。

实现步骤

findKthMin(nums1, i, nums2, j, k) 表示在 nums1[i..] 与 nums2[j..] 中找第 k 小：

  1. 某侧用完：直接返回另一侧第 k 个，即 nums[idx + k - 1]；
  2. k === 1：返回两侧首元素的较小者；
  3. 常规：两侧分别取第 k / 2 个候选，越界填 Infinity 哨兵（保证越界侧不会成为"较小候选"被误删）；比较后淘汰较小候选所在那段 half 个，指针前进 half，k -= half，递归。

复杂度

  • 时间：O(log(m + n))。
  • 空间：O(log(m + n))，递归调用栈深度。
Code
// 在 nums1[i..] 与 nums2[j..] 中找第 k 小 (k 从 1 计数)
function findKthMin(
  nums1: number[], i: number,
  nums2: number[], j: number,
  k: number,
): number {
  // 边界 1: 任一侧用完, 直接在另一侧定位
  if (i >= nums1.length) return nums2[j + k - 1];
  if (j >= nums2.length) return nums1[i + k - 1];
  // 边界 2: 取首元素较小者
  if (k === 1) return Math.min(nums1[i], nums2[j]);

  // 两侧分别取第 k/2 个候选, 越界用 Infinity 哨兵
  const half = Math.floor(k / 2);
  const val1 = (i + half - 1 < nums1.length) ? nums1[i + half - 1] : Infinity;
  const val2 = (j + half - 1 < nums2.length) ? nums2[j + half - 1] : Infinity;

  // 淘汰较小候选所在那段 half 个元素, k 同步减去 half
  if (val1 < val2) {
    return findKthMin(nums1, i + half, nums2, j, k - half);
  }
  return findKthMin(nums1, i, nums2, j + half, k - half);
}

function findMedianSortedArrays(nums1: number[], nums2: number[]): number {
  const m = nums1.length;
  const n = nums2.length;

  // 奇数: 第 (m + n + 1) / 2 小
  if ((m + n) % 2 !== 0) {
    return findKthMin(nums1, 0, nums2, 0, (m + n + 1) / 2);
  }

  // 偶数: 第 (m + n) / 2 与 (m + n) / 2 + 1 小的均值
  const num1 = findKthMin(nums1, 0, nums2, 0, (m + n) / 2);
  const num2 = findKthMin(nums1, 0, nums2, 0, (m + n) / 2 + 1);
  return (num1 + num2) / 2;
}

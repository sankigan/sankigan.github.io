673.最长递增子序列的个数

sankigan2026-7-6算法LeetCode

最长递增子序列的个数open in new window

题目描述

给定一个未排序的整数数组 nums， 返回最长递增子序列的个数。

注意：这个数列必须是 严格 递增的。

示例

输入：nums = [1,3,5,4,7]
输出：2
解释：有两个最长递增子序列，分别是 [1, 3, 4, 7] 和 [1, 3, 5, 7]。

输入：nums = [2,2,2,2,2]
输出：5
解释：最长递增子序列的长度是 1，并且存在 5 个，所以输出 5。

解答

动态规划

关键思路

本题是 300.最长递增子序列 的进阶版本，除了要求出 LIS 的长度，还要统计个数。

单个 dp 数组无法同时记录长度和方案数，因此需要维护两个数组：

  • dp[i]：以 nums[i] 结尾的最长递增子序列的长度
  • count[i]：以 nums[i] 结尾的最长递增子序列的个数

初始时两者都为 1（每个元素本身就是长度为 1 的递增子序列，方案数为 1）。

具体步骤

对于每个 i，遍历所有 j < i，若 nums[j] < nums[i]（可以接上），根据 dp[j] + 1 与 dp[i] 的关系分两种情况更新：

情况一：dp[j] + 1 > dp[i]

  • 说明找到了一条更长的递增子序列
  • 更新长度：dp[i] = dp[j] + 1
  • 更新方案数：count[i] = count[j]（继承 j 的方案数，之前的方案作废）

情况二：dp[j] + 1 === dp[i]

  • 说明找到了另一条同样长的递增子序列
  • 长度不变，方案数累加：count[i] += count[j]

情况三：dp[j] + 1 < dp[i]

  • 更短的路径，忽略

最后，累加所有 dp[i] === maxlen 的 count[i] 即为答案。

举例：nums = [1,3,5,4,7]

inums[i]分析dp[i]count[i]
01初始11
13接在 1 后：dp[0]+1=2 > 121
25接在 1 后：dp[0]+1=2 > 1，更新为 2；接在 3 后：dp[1]+1=3 > 2，更新为 331
34接在 1 后：更新为 2；接在 3 后：dp[1]+1=3 > 2，更新为 331

Code
function findNumberOfLIS(nums: number[]): number {
  // dp[i]: 以 nums[i] 结尾的 LIS 长度
  const dp = new Array(nums.length).fill(1);
  // count[i]: 以 nums[i] 结尾的 LIS 个数
  const count = new Array(nums.length).fill(1);
  let maxlen = 0;

  for (let i = 0; i < nums.length; ++i) {
    for (let j = 0; j < i; ++j) {
      // nums[i] 可以接在 nums[j] 后面
      if (nums[j] < nums[i]) {
        if (dp[j] + 1 > dp[i]) {
          // 找到更长的子序列，长度更新，方案数继承 j 的
          dp[i] = dp[j] + 1;
          count[i] = count[j];
        } else if (dp[j] + 1 === dp[i]) {
          // 找到相同长度的另一条路径，方案数累加
          count[i] += count[j];
        }
      }
    }
    // 记录全局最长长度
    maxlen = Math.max(maxlen, dp[i]);
  }

  // 累加所有长度为 maxlen 的方案数
  return count.reduce((prev, curr, idx) => dp[idx] === maxlen ? prev + curr : prev, 0);
};

易错点

  • 情况一「继承而不是累加」：找到更长的子序列时，count[i] = count[j] 而不是 count[i] += count[j]。因为之前记录的方案对应的都是更短的子序列，需要作废重新计数。
  • 情况二「累加而不是加 1」：找到相同长度的路径时，是 count[i] += count[j] 而不是 count[i] += 1。因为通过 j 可能已经有多种不同方案抵达 nums[j]，这些方案都能延伸到 nums[i]。

复杂度分析

  • 时间复杂度：O(n²)，双层循环
  • 空间复杂度：O(n)，两个 dp 数组

322.零钱兑换

sankigan2025-2-27算法LeetCode

零钱兑换open in new window

题目描述

给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。

计算并返回可以凑成总金额所需的 最少的硬币个数 。如果没有任何一种硬币组合能组成总金额，返回 -1 。

你可以认为每种硬币的数量是无限的。

示例

输入：coins = [1, 2, 5], amount = 11
输出：3

输入：coins = [2], amount = 3
输出：-1

输入：coins = [1], amount = 0
输出：0

解答

动态规划

这是一道经典的完全背包问题：硬币面额是"物品"，且每种硬币可以无限次使用，目标是用最少的"物品"凑出金额 amount。可以用一维动态规划求解。

状态定义

dp[i] 表示凑成金额 i 所需的最少硬币个数。

状态转移方程

对于金额 i，枚举每一种硬币 coin：如果 coin <= i，说明可以用一枚该硬币，将问题转化为子问题 dp[i - coin]，则：

dp[i] = min(dp[i], dp[i - coin] + 1)

即"不用这枚硬币的最优解" 与 "用一枚这枚硬币 + 凑出剩余金额的最优解" 中取较小值。

初始化

  • dp 数组长度为 amount + 1，初值设为 amount + 1（一个不可能达到的"无穷大"，因为最多用 amount 个面额为 1 的硬币就能凑成，任何真实解都不会超过这个值），代表"暂不可达"
  • dp[0] = 0：凑成金额 0 无需任何硬币

遍历顺序

外层遍历金额 i（1 → amount），内层遍历硬币面额，通过 dp[i - coin] 递推得到 dp[i]，保证子问题先于当前问题被计算。

返回结果

若 dp[amount] 仍保持初值（大于 amount），说明无法凑出该金额，返回 -1；否则返回 dp[amount]。

复杂度分析

  • 时间复杂度：O(amount * n)，n 为硬币种类数
  • 空间复杂度：O(amount)
Code
function coinChange(coins: number[], amount: number): number {
  // dp[i] 表示凑成金额 i 所需的最少硬币个数
  // 初值设为 amount + 1，作为"暂不可达"的标记
  // （最多用 amount 个面额为 1 的硬币即可凑成，真实解不会超过这个值）
  const dp: number[] = new Array(amount + 1).fill(amount + 1);
  // 凑成金额 0 无需任何硬币
  dp[0] = 0;

  // 外层遍历金额 1 ~ amount，保证计算 dp[i] 时子问题 dp[i - coin] 已经就位
  for (let i = 1; i <= amount; ++i) {
    // 内层枚举每一种硬币面额
    for (const coin of coins) {
      // 硬币面额不超过当前金额时，才能用它凑出金额 i
      if (coin <= i) {
        // 取"不用这枚硬币的最优解" 与 "用一枚该硬币 + 凑出剩余金额的最优解" 中的较小值
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }

  // dp[amount] 仍为初值，说明无法凑出该金额
  return dp[amount] > amount ? -1 : dp[amount];
}

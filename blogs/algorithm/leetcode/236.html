236.二叉树的最近公共祖先

sankigan2025-3-6算法LeetCode

二叉树的最近公共祖先open in new window

题目描述

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。

示例

输入：root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 1
输出：3

输入：root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4
输出：5

输入：root = [1,2], p = 1, q = 2
输出：1

解答

递归回溯（后序遍历）

关键思路

以 root 为根的子树中，p 和 q 的分布只有三种情况，对应三种返回值：

  • 只在左子树 → 返回左子树的递归结果
  • 只在右子树 → 返回右子树的递归结果
  • 分居左右子树（或其中一个就是 root 本身）→ 返回 root，此时 root 即为最近公共祖先（LCA）

因此可以设计一个递归函数 findNode(node)：在以 node 为根的子树中查找 p、q 的 LCA，若都不存在则返回 null。

findNode(node) 的返回值只有三种可能（这是理解后面判断逻辑的核心）：

  • 子树里 既没有 p 也没有 q → 返回 null
  • 子树里 只有 p、q 之一 → 返回那个节点本身，作为"占位"交给上层继续判断
  • 子树里 同时有 p 和 q → 返回它们的 LCA（可能在更深层就已经算出来，然后原样向上传递）

递归的三种终止/分支情况：

  1. node 为 null：子树为空，不可能包含 p 或 q，返回 null
  2. node 就是 p 或 q：根据 LCA 的定义（同时是 p、q 的祖先中最深的那个，节点可视为自己的祖先），LCA(p, q) 一定是 p 的祖先或 p 自己，绝不可能是 p 子树里比 p 更深的节点。所以无论 q 到底在不在 p 的子树中，继续往 p 的子树里递归都不会找到更优的答案——不用再往下查，直接把 node 作为候选结果返回，交给上层（node 的祖先）根据左右子树的返回值去判断真正的 LCA 在哪一层
  3. node 是普通节点：分别递归左右子树，再根据两侧结果决定当前层要返回什么
    • 左右子树都有结果 → 说明 node 就是 LCA。原因：如果 p、q 同时在左子树里，它俩的 LCA 会在左子树内部就被算出并返回，此时 p、q 已被左子树"消耗完"，右子树必然返回 null；所以左右同时非空只可能是 p、q 分居两侧（各占一侧），它们的路径只在 node 汇合，故 node 是 LCA
    • 只有一侧有结果（比如左子树非空、右子树为 null）→ 右子树为 null 说明右边一个都没找到，而题目保证 p、q 都真实存在于树中，用排除法可知它俩都在左子树，直接把左子树的结果向上传递即可，不需要额外确认
    • 两侧都没有结果 → 当前子树不包含 p 或 q，返回 null

整个过程本质上是一次后序遍历：先递归处理完左右子树，再根据子树结果决定当前节点的返回值，一旦找到 LCA 就会沿着调用栈一路返回到根节点。

注意这里依赖了题目的前提条件：p、q 都保证存在于树中且值唯一。如果不满足这个前提（例如 q 可能根本不在树里），"只有一侧有结果就直接返回"这一步就会出错。

复杂度

  • 时间复杂度：O(n)，每个节点最多被访问一次
  • 空间复杂度：O(n)，最坏情况下（树退化为链表）递归栈深度为 n
Code
function lowestCommonAncestor(root: TreeNode | null, p: TreeNode | null, q: TreeNode | null): TreeNode | null {
  // 在以 node 为根的子树中查找 p、q 的最近公共祖先
  // 若子树中不包含 p 也不包含 q，返回 null
  function findNode(node: TreeNode | null): TreeNode | null {
    // 空节点：不可能包含 p 或 q
    if (!node) return null;
    // 命中 p 或 q：交给上层判断另一个节点是否在其子树中
    if (node === p || node === q) return node;

    const left = findNode(node.left);
    const right = findNode(node.right);

    // 左右子树都找到了目标节点，说明 p、q 分居两侧，当前节点即为 LCA
    if (left && right) return node;
    // 只有左子树找到，LCA 在左子树中
    if (left) return left;
    // 只有右子树找到，LCA 在右子树中
    if (right) return right;

    // 左右子树都没找到
    return null;
  }

  return findNode(root);
}
